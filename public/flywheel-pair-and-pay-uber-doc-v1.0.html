The Problem We Solve

Street hails are analog in a digital world

When a rider flags down a Flywheel cab, they're cut off from the digital ecosystem — no tracking, no digital receipt, no app-based payment. Pair & Pay bridges that gap.

⚠️
The Problem
A rider flags down a Flywheel cab or gets dispatched without using the app. The trip is completely analog — cash or swipe only. No digital receipt. No trip history. No loyalty. No fraud protection. The rider loses all benefits of the Flywheel platform just because they didn't book ahead.
✅
The Solution
Pair & Pay lets the rider tap one button in the Flywheel app while already in the cab. A secure 4-digit code links their session to the driver's tablet. The trip is confirmed, tracked, and charged automatically — no cash, no card swipe, no pre-booking required.

Technical Handshake

How the pairing session works

The pairing is a 3-way secure handshake between the Rider App, the Flywheel Server, and the Driver Tablet — ensuring the correct trip, driver, and payment are always matched.

Pair & Pay Session Handshake Flow
1
Rider taps Pair & Pay → Rider App sends session open request to Flywheel Server → Server generates unique 4-digit code tied to rider session ID + timestamp
2
Driver taps Enter PIN → Driver Tablet enters ready-to-pair state → Code is transmitted from rider verbally (out-of-band)
3
Driver submits code → Driver Tablet sends code + driver session ID to Flywheel Server → Server validates: code match + time window + no prior use
4
Server returns trip context to Rider App (driver name, photo, vehicle, plate, pickup/dropoff) → Rider reviews and confirms → Server locks the pairing
5
Both devices receive pairing confirmation → Meter fare is linked to rider's payment token → At trip end, server auto-processes charge and sends receipt to rider
🔐 Code is single-use
⏱️ Session window expires
✓ 5-attempt fraud limit
✓ Rider verifies driver before payment links
⚡ Full handshake < 30 seconds

🚖

Drivers receive guaranteed digital payment — no cash disputes, no change-making, no terminal swipes. Payment confirmed before drop-off. Applies to street hails and dispatched trips equally.

Digital receipt, trip history, loyalty points, and saved payment — even on a street hail. Riders no longer lose platform benefits just because they didn't pre-book. Identity and payment are always protected.
🏢
For Fleet Operators
Full trip visibility & digital revenue
Every paired trip is captured in Flywheel's system — full audit trail, revenue reconciliation, and driver performance data. Eliminates the cash blind spot in street hail revenue.
🤝

Riders who would otherwise use cash or swipe are now in a digital ecosystem — trackable, retargetable, and potentially convertible to Uber platform trips. Pair & Pay is a bridge into the Flywheel × Uber network.

Edge Cases & Fallbacks

Graceful degradation at every failure point

Pair & Pay is designed to never disrupt the trip. If pairing fails for any reason, the ride continues normally.

🔢
Invalid Code
Wrong PIN entered
Driver sees "Invalid code" with remaining attempts displayed. Up to 5 attempts allowed. On exhaustion, pairing is blocked and driver reverts to standard cash/card collection.
📶
Network Issue
Connection lost mid-pairing
"Pairing interrupted" dialog appears with a Try Again option. If the session is still active, driver can retry. If unrecoverable, reverts to standard collection with no trip disruption.
❌
Rider Cancels
Rider doesn't confirm
"Pairing cancelled" message shown to driver. Trip continues with standard payment. Neither driver nor rider is penalized — the meter has been running throughout.
