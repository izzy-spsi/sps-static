How to Install a New Skill
1

Find on ClawHub

Ask Jarvis to search ClawHub: "find a skill that does X" — uses the clawhub skill to search the registry at clawhub.ai

2

Install via CLI

openclaw skills install <skill-name>

or ask Jarvis to install it using the clawhub skill

3

Enable in config

Skills are auto-enabled on install. To disable: add to skills.entries in openclaw.json with enabled: false

4

Document it here

Update this portal's Skills tab and add any local config notes to TOOLS.md in the workspace

Commands Reference
Slash commands and one-word commands that Izzy uses to direct Jarvis. All agents must know these.
⚡ Slash Commands
/draft
Write and show ONLY. Never send. Izzy decides what happens next. Even if the draft looks ready — NEVER send unless explicitly told to.
/send
Send immediately, no confirmation needed.
/note
Log to memory only, no response shown.
/revise
Update the last draft with new direction provided.
/shorter
Make the last response or draft shorter.
/longer
Make the last response or draft longer.
/formal
Adjust tone to formal/professional.
/casual
Adjust tone to casual/conversational.
/commit
Save session to all places: memory (YYYY-MM-DD.md + MEMORY.md) + Notion + Jarvis HQ Dashboard + git tag.
/recall [topic]
Retrieve everything about a topic into context: search MEMORY.md + daily logs + Notion + Google Workspace (Gmail/Drive/Calendar) + Jarvis HQ Dashboard. Compile into a clean context summary.
One-Word Commands
Commit / Commit All
Same as /commit — save to memory + update Jarvis HQ Dashboard + update Notion + git tag. Confirm when all four done.
Monitor
Set up 5-min cron QA loop → auto-fix → auto-stop when clean → alert Izzy. Remove cron automatically when all clear.
Note Mode
Silent intake — receive notes only, no building or questions.
Planning Mode
Open discussion / Q&A — ask questions, explore ideas, shape direction together.
Compile
Synthesize all notes + discussion into a clean spec.
Build
Execute using compiled spec.
Workflow Flows
Build Flow
Note Mode → Planning Mode → Compile → Build → Commit
Draft Flow
/draft → [Izzy reviews] → /revise OR /send
Context Flow
/recall [topic] → work → /commit

⛔

groupPolicy is "open" only for monitoring. Not for responding. @mentions → alert Izzy + draft → wait for approval.
⛔
Never run or report health checks proactively
Never send "all clear" or "all systems healthy" messages ever. Only alert if something is actually broken. Established 2026-05-12.
⚠️
Every PDF must be saved locally AND uploaded to Google Drive
Local: /workspace/storage/ | Drive: account matching the project. Always return Drive link after upload. No exceptions. Established 2026-05-28.
⚠️
Bot Delegation Rule
When a task belongs to a business bot (JarvisSPS, JarvisDomo, etc.) — relay, don't execute. Step in ONLY if the business bot explicitly escalates. Main Jarvis = Chief of Staff. Business bots = operators. Violated 2026-05-10.
⚠️
Draft = show only. Never send.
The word "draft" alone is never a send instruction. Even if the draft looks ready. Only send when Izzy explicitly says "send", "send it", or /send.
⚠️
Versioning: X.Y.Z — Two-place update rule
Z = patch (bug fixes, increment forever). Y = minor (confirmed working, reset Z). X = major milestone. Always update BOTH package.json AND hardcoded UI version string. Established 2026-05-14.
💡
trash > rm
Recoverable beats gone forever. Always use trash command instead of rm for file deletions.
💡
MEMORY.md is main session only
Contains personal context. Do NOT load in shared contexts (Discord, group chats, sessions with other people). Security measure.

Light / Dark Mode Toggle
Top right corner button. Dark: Carbon bg, Warm White text, accent color. Light: Warm White bg, Carbon text. Photo-backed slides always keep white text.
🖨️
Print to PDF Button
Top right, always forces Light Mode for printing. @media print: landscape, no nav, all slides/pages paginated.
📊
Real-Time Calculations (where applicable)
Financial/simulator portals: all inputs trigger instant recalc via oninput events. No "calculate" button needed. Chart.js for charts with maintainAspectRatio:false.
📱
Self-Contained HTML
All assets embedded (fonts via CDN, images as base64 when needed). Single file deployable anywhere. No external dependencies except Google Fonts + Chart.js CDN.

SPS S-mark
Bottom right corner on all SPS-branded portals. Omit only when explicitly told to.

Cost Optimization Strategies
💰
Haiku for heartbeats and cron
Every 30-min heartbeat uses Haiku (~60x cheaper than Sonnet). All cron jobs default to Haiku unless complex reasoning is required.
💰
Cache TTL: 1 hour
Context is cached and reused within the hour. Reduces redundant token spend on repeated context loading.
💰
Bootstrap max: 40K chars (80K total)
Limits how much context is loaded at session start. Keeps costs manageable for long-running sessions.
💰
Railway for dynamic, GitHub Pages for static
Static portals (no server-side logic) go to GitHub Pages (free). Only Railway for apps that need a server.
