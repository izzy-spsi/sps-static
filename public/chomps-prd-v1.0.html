01 — Overview

What is Chomps?

Chomps is Domo Cafe's fan loyalty platform — not a punch card, but a brand membership system. Named after Domo-kun's iconic open-mouth "chomp," it rewards guests for every interaction with the brand: visiting the cafe, buying merchandise, engaging on social media, and sharing their love of Domo with others.


Unlike traditional restaurant loyalty programs, Chomps works globally. A fan in Tokyo who can't visit Buena Park can still earn Chomps by following on Instagram, buying merch online, or completing digital challenges. When the online store launches, Chomps is the currency that connects the entire ecosystem.

🍽️
In-Cafe
Earn Chomps on every visit. QR code on table tent or receipt → chomps.domokuncafe.com → phone number ties the order to your account.
📱
Social
Follow, post, tag, and share for Chomps. Every piece of user-generated content grows the brand and earns the fan — no cafe visit required.
🌐
Online
Sign up, refer friends, buy from the online merch store. Global fans earn Chomps and build toward rewards before the store even launches.
Strategic purpose beyond loyalty: Chomps is the identity and data layer for the entire Domo Cafe intelligence system. Every member ties a real person to every purchase, social action, and visit — powering the forecasting model, measuring marketing impact, tracking return rates, and feeding the scenario engine.

WhoRiddle Partnership: Users connect their WhoRiddle account to Chomps. Every correct riddle answer earns 100 Chomps. Daily streaks and perfect weeks trigger bonus multipliers. WhoRiddle fires a webhook → Chomps credits the linked account. No manual steps for the guest.
Dynamic Multipliers: Off-peak hours (weekday lunch, slow periods) automatically trigger bonus earn rates to drive traffic. Example: Monday 11am–2pm → 2x Chomps on all F&B. Triggered by Toast occupancy data — tables below 40% → multiplier activates.
03 — Redemption Model

Milestone Rewards + Selective Redemption

Chomps works two ways: milestone rewards (automatic surprises at key thresholds) and selective redemption (guest chooses specific items). Milestones create emotional moments — redemption gives flexibility. Both together match the Domo brand: fun, surprising, and rewarding.

05 — Waitlist & Reservations

From Walk-In to Valued Guest

Currently walk-in only with no waitlist. A digital waitlist solves three problems at once: guest experience, Chomps enrollment, and operational data.
The enrollment moment: A guest waiting for a table has their phone in hand, nothing to do, and high motivation. This is the highest-converting Chomps enrollment window you'll ever have. Guest scans door QR → joins waitlist with phone number → immediately offered Chomps signup → 500 Chomps on today's visit if they sign up now.
🔢 Digital Waitlist (Phase 2)
Guest scans QR at entrance → enters phone → joins waitlist → gets SMS when table is ready. Frees guests to browse merch, explore, or sit outside instead of standing in line. Captures every phone number for Chomps enrollment.
📅 Reservations (Chomps Unlock)
  • Domo Friend — Waitlist only
  • Domo Fan — Reserve 2 seats, 48hr advance
  • Domo Obsessed — Reserve 6 seats, 72hr advance
  • Domo Legend — Priority table, any time
30% of tables reserved · 70% walk-in always protected

10 — Data Model

Core Data Schema

PostgreSQL via Prisma. Every table feeds both the loyalty system and the forecasting engine.
-- Core member identity members { id UUID PRIMARY KEY phone VARCHAR UNIQUE -- primary in-cafe identity email VARCHAR UNIQUE -- for remote fans name VARCHAR birthdate DATE chomps_balance INTEGER DEFAULT 0 -- current spendable balance chomps_lifetime INTEGER DEFAULT 0 -- never decrements — used for tier tier ENUM(friend, fan, obsessed, legend) referral_code VARCHAR UNIQUE referred_by UUID REFERENCES members created_at TIMESTAMPTZ last_activity TIMESTAMPTZ } -- Every Chomps transaction (earn or spend) transactions { id UUID PRIMARY KEY member_id UUID REFERENCES members type ENUM(earn, redeem, expire, adjust) source ENUM(purchase_fb, purchase_merch, social_follow, social_tag, social_tiktok, referral, signup, survey, birthday, online_store, manual, milestone) chomps INTEGER -- positive=earn, negative=spend reference_id VARCHAR -- Toast order ID, post URL, etc. metadata JSONB created_at TIMESTAMPTZ } -- In-cafe visits — links Toast order to Chomps member visits { id UUID PRIMARY KEY member_id UUID REFERENCES members toast_order_id VARCHAR date DATE daypart ENUM(lunch, dinner) fb_revenue DECIMAL(10,2) merch_revenue DECIMAL(10,2) covers INTEGER chomps_earned INTEGER multiplier DECIMAL(4,2) DEFAULT 1.0 } -- Social platform connections social_connections { id UUID PRIMARY KEY member_id UUID REFERENCES members platform ENUM(instagram, tiktok, yelp, google) handle VARCHAR platform_id VARCHAR -- platform's user ID for webhook matching access_token TEXT ENCRYPTED follows_domo BOOLEAN verified_at TIMESTAMPTZ } -- Digital waitlist waitlist { id UUID PRIMARY KEY phone VARCHAR member_id UUID REFERENCES members NULLABLE party_size INTEGER position INTEGER status ENUM(waiting, seated, left, no_show) joined_at TIMESTAMPTZ seated_at TIMESTAMPTZ wait_minutes INTEGER -- actual wait time — feeds forecast }
Forecasting bridge: The visits table links every Chomps member to real revenue, daypart, and covers. The waitlist table captures actual wait times and party sizes, feeding projected cover accuracy. Together they transform anonymous Toast data into attributed guest intelligence.
