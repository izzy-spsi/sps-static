06 — Gap Analysis
What's missing before this is a complete workflow.
A.I.R. Provisioning System
No system exists to spin up isolated OpenClaw instances for members. This is the entire infrastructure foundation of A.I.R. — it doesn't exist yet.
Impact: Critical — blocks all of A.I.R.
Member Naming Flow
"Name your Regent" onboarding moment doesn't exist. This is the first personalization step — sets IDENTITY.md, SOUL.md template, and Telegram bot name.
Impact: High — core UX moment
Manifest System
The Living PRD system is fully defined but not built. No requirement tracking, no impact engine, no changelog, no access control.
Impact: High — blocks all PRD work
Manifest Agent
The agent that converts brainstorm → structured PRD and manages changes with impact analysis. Not defined in agent spec.
Impact: High — core Manifest function
UX/UI Agent
No agent defined for wireframes, screen flows, or design system generation. UI changes need to link back to Manifest requirements.
Impact: Medium — needed for Build stage
QA / Test Agent
No agent generates test scripts from requirements or links test results back to Manifest. Core to the Living PRD concept.
Impact: Medium — needed for Manifest loop
Change Governance
Who approves changes is defined (project owner). But the notification flow, approval UI, and rejection workflow are not designed.
Impact: Medium — needed for Manifest
API Key Transition
SPS manages keys at first → BYO later. The transition mechanism, key storage security, and OpenRouter integration are not designed.
Impact: Medium — needed before launch
Portability / Migration
$1,500 migration fee is set. But the export format, migration tooling, and re-import process for A.I.R. instances are not designed.
Impact: Medium — needed at launch
Financial Agent
Accounting, tax, CPA referral, and burn rate monitoring are undefined. Critical for Stage 5–6 members.
Impact: Medium — Stage 5 blocker
Growth Agent
GTM execution, marketing, KPI tracking, hiring plan — Stage 7 is entirely undefined. No agent covers post-launch scaling.
Impact: Low now — future critical
GitHub / Code Integration
Commits linked to REQ IDs is designed for v2. The data model needs to support it from v1. Not yet in Manifest spec.
Impact: Low for v1 — design now
