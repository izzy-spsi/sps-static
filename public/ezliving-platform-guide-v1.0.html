3
How an Investor Gets Access

This is the flow every investor goes through. You only step in at Step 4 — everything else is automated.

1

Investor fills out the contact form

They visit ezliving-portal-production.up.railway.app (soon: ezliving.com), enter their name, email, phone, and interest level.

2

NDA signing via DocuSeal

Immediately after submitting, they're directed to sign the EZLiving Investor NDA. The document is pre-filled with EZLiving LLC / Tim Ziss on the company side — investor only needs to fill their name, title, and signature.

The signed NDA is stored automatically. You'll receive an email alert at izzy@spsi.ai when an investor completes signing.

3

Investor lands on confirmation page

They see a confirmation that their NDA was received and that an access code will be sent shortly.

4

You generate and send the access code (1 click)

Go to the Admin Panel → find the investor in the leads list → click "Generate Code" → click "Send Code". The investor receives an email with their unique access code automatically.

This is the only manual step in the entire flow. A future update will automate this on NDA completion.

5

Investor enters their code → full materials access

They return to the portal, enter their unique access code, and immediately see all four investor decks. Each deck opens in a new tab. If they try to access a deck directly without a valid code, they're redirected back to the portal to request access.

Security Overview

How the platform protects the decks from unauthorized access.

Layer How It Works
NDA Gate Investor must sign before receiving an access code. Signed documents stored in DocuSeal.
Access Code Unique per-investor code. Required to unlock the materials page.
Deck Token Materials page injects a session token into every deck link. Token is stored in the browser and verified on each deck load.
Direct URL block Anyone who navigates directly to a deck URL without a valid token is immediately redirected to the investor portal.
Admin bypass Separate admin key — works independently from investor tokens. Can be rotated at any time without affecting investor access.
